# Minerva Next Practical Value Path Decision Point v0.1

## Decision Required

Choose the next practical Minerva value path after completing the Developer Log durable-evidence controlled-answer path.

## Option A: Add Hardening Log as Second Durable Evidence Type

Extend the practical durable-evidence pattern to Hardening Log evidence. This would test whether the local fixture, rollback, retrieval-readiness, answer-preparation, rehearsal, review, and boundary controls generalise beyond Developer Log.

## Option B: Add Platform Doctrine as Second Durable Evidence Type

Extend the practical durable-evidence pattern to Platform Doctrine evidence. This would test higher-level decision and operating-model evidence while preserving strict source and status boundaries.

## Option C: Controlled Retrieval Harness Over Durable Evidence Fixtures

Create a deterministic harness that reads controlled fixture metadata and evaluates retrieval eligibility and answer-preparation readiness without live retrieval, DB access, chat, or LLM calls.

## Option D: Pause Minerva While Analytics Schema Work Continues

Pause Minerva expansion and keep the completed Developer Log path as the current durable reference while analytics schema work continues elsewhere.

## Recommended Default Next Step

Option C is the recommended default because it adds practical value over the completed Developer Log path while preserving local deterministic boundaries.

## Preconditions For Each Option

Option A requires a selected Hardening Log source and explicit durable-evidence candidate criteria.

Option B requires a selected Platform Doctrine source and explicit rules for doctrine status, supersession, and source boundaries.

Option C requires confirmation that only checked-in local fixtures may be read and that no live retrieval backend, DB, chat, or LLM path is authorised.

Option D requires an explicit pause decision and a handoff note identifying Minerva as complete for the current practical path only.

## What Must Not Be Done Without Explicit Authorisation

Do not call a live LLM, generate final user-facing answers, expose chat, register routes, connect to a DB, read or write a DB, mutate a live corpus, ingest Code Evidence, change the retrieval backend, change UI, or claim runtime, deployment, or production readiness.

## Developer Handoff

Use this decision point to select the next slice. The completed Developer Log path is local deterministic metadata only and must remain bounded until a separate authorisation expands the scope.
